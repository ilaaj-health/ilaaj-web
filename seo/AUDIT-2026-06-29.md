## Fixes shipped this run

### ✅ #45 — Doctor structured-data markup errors (105) → [PR #52](https://github.com/ilaaj-health/ilaaj-web/pull/52)
`/doctors/*` and `/pk/doctors/*` emitted JSON-LD `@type: "Physician"`. Schema.org makes
`Physician` a subtype of both **LocalBusiness** (requires a postal `address` — online-only
doctors have none) and **Organization** (where `worksFor` is invalid). That produced 2 errors ×
~52 doctor pages ≈ 105. Re-modeled each doctor as a **`Person`** (valid `worksFor`, no required
address; `jobTitle`/`knowsAbout` preserve specialty). Files:
`src/pages/pk/doctors/[slug].astro`, `src/pages/doctors/[slug].astro`. Build verified (303 pages).

### ✅ #204 — Hreflang language mismatch (17) → [PR #53](https://github.com/ilaaj-health/ilaaj-web/pull/53)
All pages declared `hreflang="ur-PK"` for the `/pk/*` URLs, but that section serves
Roman-Urdu/English copy that Google detects as `en`. Changed the `/pk/` alternate to **`en-PK`**
in `src/components/SEO.astro`; true Urdu-script content at `/ur/*` keeps `hreflang="ur"`.
Build verified.

### ✅ #4 — Blocked from crawling (11) — diagnosed, no change
`robots.txt` is fully permissive (`Allow: /`) and `vercel.json` sets a global
`X-Robots-Tag: index, follow`. The crawl hit its 100-page budget (`isPagesLimitReached: true`),
so the 11 "blocked" pages are a crawl-budget artifact, not a code defect. **Recommendation:**
raise the Semrush crawl page limit; no code change made (loosening permissive config on a guess
would be wrong).
